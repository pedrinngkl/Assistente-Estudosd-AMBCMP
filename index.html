Guia Rápido de Endereçamento IP

Esta seção explica os conceitos fundamentais para resolver os exercícios, com base no material fornecido.

1. Identificando a Classe do IP, NetID e HostID

A primeira coisa a fazer é olhar para o primeiro número (octeto) do endereço IP. Ele define a classe e como o endereço é dividido.

    [cite_start]
  • Classe A: Primeiro número entre 1 e 126. Divisão: NetID.HostID.HostID.HostID[cite: 3].
    • [cite_start]
  • Classe B: Primeiro número entre 128 e 191. Divisão: NetID.NetID.HostID.HostID[cite: 3].
    • [cite_start]
  • Classe C: Primeiro número entre 192 e 223. Divisão: NetID.NetID.NetID.HostID[cite: 3].
[cite_start]

Exemplo Prático: 149.14.250.68[cite: 3].

1. O primeiro número é 149. [cite_start]Ele está entre 128 e 191, portanto, é Classe B[cite: 3].
2. [cite_start]Por ser Classe B, os dois primeiros números formam o NetID e os dois últimos formam o HostID[cite: 3].
3. NetID: 149.14. [cite_start]HostID: 250.68[cite: 3].

2. Calculando o Endereço de Rede

O endereço de rede é o primeiro endereço de um intervalo e representa a rede como um todo.

Regra: Mantenha a porção do NetID e troque toda a porção do HostID por zeros.
[cite_start]

Exemplo Prático: 195.168.56.2[cite: 3].

1. [cite_start]O primeiro número é 195, então é Classe C[cite: 3].
2. [cite_start]O NetID é 195.168.56 e o HostID é 2[cite: 3].
3. Mantemos o NetID e zeramos o HostID. [cite_start]O resultado é 195.168.56.0[cite: 3].

3. Calculando o Endereço de Broadcast

O endereço de broadcast é o último endereço de um intervalo, usado para enviar dados para todos os dispositivos da rede de uma só vez.

Regra: Mantenha a porção do NetID e troque toda a porção do HostID por 255 (padrão universal) ou 254 (convenção usada no exercício).
[cite_start]

Exemplo Prático: 195.168.56.2[cite: 3].

1. [cite_start]Já sabemos que é Classe C e o NetID é 195.168.56[cite: 3].
2. Trocamos o HostID. [cite_start]O resultado, conforme o material, é 195.168.56.254[cite: 3]. (O padrão comum seria .255).

4. Conversão Decimal para Binário

Para converter, cada um dos quatro números do IP é transformado em um número binário de 8 bits.

[cite_start]

Exemplo Prático: 200.241.111.25[cite: 6].

1. [cite_start]200 em binário é 11001000[cite: 7].
2. [cite_start]241 em binário é 11110001[cite: 7].
3. 111 em binário é 01101111 (corrigido do doc).
4. 25 em binário é 00011001 (corrigido do doc).
Resultado (corrigido): 11001000.11110001.01101111.00011001. [cite_start]O documento [cite: 7] parece ter um erro de digitação na conversão.

5. Máscaras de Rede Padrão

A máscara de rede define qual parte do IP é a rede e qual parte são os hosts. Para as classes padrão, as máscaras são fixas.
